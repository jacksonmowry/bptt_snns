#+title: TENNLab Backpropagation Through Time Demo
#+author: Jackson Mowry

This is a work in progress demoing how we can train "hairball" spiking neural networks with backpropagation through time.

* Usage
** Cloning
=git clone --recurse-submodules https://github.com/jacksonmowry/bptt_snns=
** Building
Just run =make=
** Datasets
Either supply your own, or run the =scripts/datasets.py= script to fetch a common dataset.
** Running
You'll need both data/label CSVs, the application will parse everything for you.

#+begin_src shell
bin/bptt_learning --network_json networks/risp_f_emptynet.json --data_file mnist_data.csv --label_file mnist_label.csv --learning_rate 0.0084 --decay_rate 0.0001 --tau 0.95 --rho 1.4 --timesteps 32 --hidden_neurons 64 --epochs 1000 --batch_size 10 --connectivity 0.1
#+end_src

* Resuming Training
** Pre-train a network
Save the trained network with =--network_json_out=. This writes the full network JSON with metadata embedded in =Associated_Data -> other=.

#+begin_src shell
bin/bptt_learning --network_json networks/risp_f_emptynet.json \
  --data_file mnist_data.csv --label_file mnist_label.csv \
  --learning_rate 0.0084 --decay_rate 0.0001 --tau 0.95 --rho 1.4 \
  --timesteps 32 --hidden_neurons 64 --epochs 100 --batch_size 10 \
  --connectivity 0.1 \
  --network_json_out models/checked_network.json
#+end_src

** Reload and continue training
Use the same =--network_json= pointing to a saved network. When the network has nodes/edges and embedded metadata, all CLI arguments are overridden by the saved values. The program prints a warning listing every overridden parameter. You only need to provide the network file and data — no hyperparameters required.

#+begin_src shell
bin/bptt_learning --network_json models/checked_network.json \
  --data_file mnist_data.csv --label_file mnist_label.csv
#+end_src

If the network is empty (no nodes/edges), it generates a fresh network from scratch using the default CLI parameters. If the network has structure but no metadata, it warns and falls back to defaults:

* Testing

Golden output testing ensures deterministic reproducibility — same inputs and seed
produce byte-identical stdout and trained network across builds.

** Running Tests

#+begin_src shell
./tests/run_golden_tests.sh
#+end_src

Runs all tests in =tests/golden/= and compares stdout + network JSON against
golden baselines. Only =compile_time= and =start_time= are stripped from network
comparison — all other metadata is compared verbatim.

** Regenerating Golden Files

#+begin_src shell
./tests/update_golden.sh [test_name]
#+end_src

Without arguments, regenerates golden files for all tests. With a test name
(full =golden_test_N_xxx= or short =N_xxx=), regenerates only that test.
This overwrites existing =.out= and =.net.json= files in-place.

** Adding a Test

  1. Create =tests/golden/N_<name>= directory
  2. Add =golden_test_N_<name>.cfg= with =key=value= CLI arguments
  3. Run =tests/update_golden.sh golden_test_N_<name>= to generate golden files
  4. Run =tests/run_golden_tests.sh= to verify they match

See =tests/golden_tests.org= for full documentation and directory structure.

* Network Generation
Networks are create with the following parameters
- Input neurons equal to =2 * data_columns=
- Hidden neurons equal to =--hidden_neurons=
- Output neurons equal to =len(unique(labels))=
- Synapses are generated between any 2 neurons (including self connections) with probability =--connectivity=
  - Synapses take on a random delay between =1= and =7=
  - Synapse weights are initialized to small random values normal(0.0, 0.1) for floating-point, and a scaled version for discrete
  - Floating-point synaptic weights are currently bound to [-1.0, 1.0] to match discrete network behavior, through this behavior should likely be removed

* Network Training
- Currently only synaptic weights are trained
- Loss values are calculated from categorical cross-entropy, utilizing spike rate as the decoded value between [0.0, 1.0] 
- Input values are encoded on 2 neurons, with spikes mirroring each other

* Training A Discrete (integer only) SNN
Make the app.
#+begin_src shell
make CFLAGS='-O3'
#+end_src

Download a dataset.
#+begin_src shell
python -m venv venv
pip install pandas numpy scikit-learn
python scripts/datasets.py -d data/ breast_cancer
deactivate
#+end_src

Train your network. The app automatically detects if you're training a discrete network (based on the =--network_json= parameter) and will enable quantization based on the =min_weight= and =max_weight= parameters.
#+begin_src shell
bin/bptt_learning \
    --network_json networks/risp_127.json \
    --data_file data/breast_cancer_data.csv \
    --label_file data/breast_cancer_label.csv \
    --timesteps 32 \
    --hidden_neurons 16 \
    --connectivity 0.2 \
    --epochs 1000 \
    --batch_size 100 \
    --threads 4 \
    --decay_rate 0.01 \
    --network_json_out networks/breast_cancer.json
#+end_src

* Running Networks in Interactive Mode
To run a network with the =processor_tool_risp= binary we can do the following.

Train a network.
#+begin_src shell
$ bin/bptt_learning --network_json networks/risp_f_emptynet.json --data_file data/iris_data.csv --label_file data/iris_label.csv --hidden_neurons 4 --connectivity 0.75 --timesteps 16 --training_percent 0.75 --epochs 100 --threads 10 --batch_size 10 --network_json_out iris_test.json

We then run the network through =bin/interactive_mode= which accepts observations as space separated values on =stdin=. The flag =--test= specifies if we should use the training dataset min/max values for encoding, or the test values.
#+begin_src shell
$ head -1 data/iris_data.csv | sed 's/,/ /g' | bin/interactive_mode --test --network_json iris_test.json

This produces the interactive commands you'd typically type into the processor tool. Now we can pipe this into the processor tool.
#+begin_src shell
$ head -1 data/iris_data.csv | sed 's/,/ /g' | bin/interactive_mode --test --network_json iris_test.json | framework-open/bin/processor_tool_risp 
node 12 spike counts: 14
node 13 spike counts: 0
node 14 spike counts: 1

$ tail -1 data/iris_data.csv | sed 's/,/ /g' | bin/interactive_mode --test --network_json iris_test.json | framework-open/bin/processor_tool_risp 
node 12 spike counts: 1
node 13 spike counts: 1
node 14 spike counts: 13
#+end_src

We see the most spikes on the first output for the first observation and the last output for the last observation. Looking at the labels for these 2 examples we can confirm that we are spiking for the correct class in each case.

#+begin_src shell
$ head -1 data/iris_label.csv 
0
$ tail -1 data/iris_label.csv 
2
#+end_src
